MINI CURSO WORDPRESS

Introdução

O Wordpress.org deve ser instalado em uma hospedagem própria e usar um domínio próprio para ficar disponível.

As vantagens do WordPress instalado são inúmeras, e a melhor vantagem é que não existem restrições para instalar temas e plugins. Ou seja, é possível desenvolver um tema ou plugin para suprir as necessidades do site.




Instalação

Primeiramente, baixe a versão mais recente do WordPress em português do Brasil neste link.

Após fazer o download, extraia os arquivos para o diretório que o WordPress ficará instalado.

O próximo passo é criar uma base de dados. O WordPress usava somente banco de dados MySQL.

Criando um banco de dados MySQL localmente

O XAMPP possui o MySQL instalado. No painel do XAMPP, inicie o MySQL se ele não estiver iniciado.

Após iniciado, vá para http://localhost:8080/phpmyadmin/ ou http://localhost/phpmyadmin/.

Clique em Utizadores na barra de menu do topo, em seguida Adicionar utilizador. Preencha todas as informações de login e atente-se aos três itens:

  • Na opção Máquina, selecione local;
  • Na opção Database for user, selecione “Create database with same name and grant all privileges”;
  • Na opção Privilégios Globais, selecione Todos.

Instalando o WordPress

Criado o banco de dados, acesse pelo navegador o endereço com o nome do diretório, no meu exemplo é http://localhost:8080/wordpress/.

Agora você está pronto para a “famosa instalação de 5 minutos”. Avance a tela até chegar na parte de preencher os dados.

Primeiro conecte-se ao banco de dados utilizando as informações de login criadas anteriormente.

Por fim, preencha informações refente ao login do administrador do site e informações do site.

Você acabou de terminar a “famosa instalação de 5 minutos” do WordPress que não leva metade desse tempo.




Personalização

Agora o site está funcionado e tudo está ok. O próximo passo é desenvolver um tema próprio, instalar os plugins necessários ou até fazer seu próprio plugin.

Temas

Para começar um tema para o Wordpress você precisa, de pelo menos, a seguinte estrutura:

  • index.php (Modelo da página inicial)
  • header.php (Modelo do topo, sempre onde a tag head estará para definir as meta, estilos, importações)
  • sidebar.php (Modelo geralmente para a coluna lateral; É possível também habilitar os Widgets que possibilitarão adicionar coisas à lateral sem usar código e encher este template com linhas de código manualmente)
  • footer.php (Modelo que finalizará todo o HTML)
  • single.php (Modelo usado para exibir a postagem)
  • page.php (Modelo usado para exibir uma página)
  • functions.php (Todas as funções PHP que o tema usa devem estar aqui, por exemplo, a função para habilitar thumbnails nos posts)
  • style.css (Estilo principal do tema)

Há outros modelos de templates que são usados para complementar um tema, tais como comments.php (serve para o modelo dos comentários nos posts e páginas), archive.php (modelo para mostrar de forma diferenciada os resultados de uma categoria, tag, post por autor), 404.php (modelo de página de ERROR 404 personalizada), entre outros.

Existem vários temas prontos e temas base (para começar praticamente do zero). Atualmente, indico o Underscores, pois já está usando o HTML5, tem comentários nos templates (em inglês) e está sob licença GPL (Licença Pública Geral).

O Underscores é um tema para ser usado como tema pai, por isso devemos criar um tema filho para modificá-lo e não mexer em seu “núcleo”.

Temas Filhos

O tema filho é um tema que herda a funcionalidade de um outro, chamado tema pai, que permite modificar e/ou acrescentar funcionalidades e estilos do tema principal sem modificá-lo.

Criar um tema filho é a maneira mais recomendada de fazer modificações em um tema.

Para criar um tema filho, crie um diretório novo. É recomendado que você acrescente “-child” no final do diretório, principalmente para organização. Por exemplo: underscores-child.

O que diferenciará o tema filho do tema pai será o arquivo style.css, que deverá conter:

/*
Theme Name:     Underscores Child
Theme URI:      http://underscores.me/
Author:         Automattic
Author URI:     http://automattic.com/
Version:        1.0.0
Template:       underscores
*/

@import url("../underscores/style.css");

É importante indicar o template pai e fazer a importação do estilo do tema pai e que esta importação fique acima de todo o código CSS que será inserido no arquivo (para não anular a importação do estilo do tema pai).

Agora é só acrescentar novas funções, editar ou alterar templates e depois de tudo pronto começar a publicar conteúdos no site.
